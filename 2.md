> [!NOTE]
> **Cloche OS is a headless base image**, not a desktop distribution. It is intended as a foundation for building custom immutable systems via BlueBuild, rpm-ostree, or bootc workflows.

---

## Why Cloche?

- **Declarative by design**: Define your entire OS state in YAML, trackable via Git.
- **Strong defaults**: SELinux enforced, Cockpit enabled, Tailscale ready — secure and manageable out of the box.
- **Batteries included**: Useful tools (`distrobox`, `just`, `tmux`, `btop`).
- **Flexible downstreams**: Use as a base for Cloche Standard Desktop Series (`GNOME` and `Plasma`), CI runners, or edge appliances.

---

## Core Architecture

| Component | Details |
|-----------|---------|
| **Base** | Universal Blue (`ublue-os/base-main`) |
| **Build System** | BlueBuild framework + GitHub Actions |
| **Package Model** | rpm-ostree (image-based, atomic updates) |
| **Container Runtime** | Podman + Docker (rootless/rootful ready) |
| **Security** | SELinux enforced, signed images via Sigstore/Cosign |
| **Remote Management** | Cockpit socket enabled by default |

---

## Key Features

- **Minimal Footprint**: No GUI, no Wayland, no desktop codecs — just the essentials.
- **Predictable Lifecycle**: Atomic updates with rollback via `rpm-ostree`.
- **Infrastructure-Ready**: `distrobox`, `just`, `tailscale`, `cockpit`, and networking tools pre-installed.
- **GitOps Friendly**: Entire OS definition lives in YAML; rebase, test, and promote like application code.
- **Secure by Default**: SELinux profiles, signed images, and Secure Boot support out of the box.

## Deployment & Installation

### Remote Rebase (from compatible Atomic host)

```bash
# Rebase to Cloche OS headless base
rpm-ostree rebase ostree-unverified-registry:ghcr.io/cloche-project/cloche:latest

# Apply changes
systemctl reboot
```
### Post-Rebase Verification

```bash
# Confirm active deployment
rpm-ostree status

# Quick system overview
fastfetch  # or use 'htop','btop', etc.

# Verify Cockpit is accessible
systemctl status cockpit.socket
# Then visit https://<your-host>:9090
``` 
## Building Your Own Downstream

Cloche is designed to be extended. Use BlueBuild to create your custom variant:

```bash
# Clone the BlueBuild template
git clone https://github.com/ublue-os/bluebuild
cd bluebuild

# Point to Cloche as your base in recipe.yml:
# base-image: ghcr.io/cloche-project/cloche:latest

# Build your custom image
just build
``` 
## Verification & Security
Images are signed with Sigstore Cosign. Verify before deployment:

```bash
# Download the public key from the repository
curl -O https://raw.githubusercontent.com/cloche-project/cloche/main/cosign.pub

# Verify the image signature
cosign verify --key cosign.pub ghcr.io/cloche-project/cloche:latest
```

>[!IMPORTANT]
Secure Boot Users: Enroll the public key (secure_boot.der) before rebasing:
```bash
sudo mokutil --import secure_boot.der
# Reboot and follow MOK enrollment prompts
# Default password: universalblue
```

## Contributing & Building Your Own
Cloche is built entirely via GitHub Actions. To create a custom variant:

1. Fork cloche-project/cloche
2. Add your signing key as SIGNING_SECRET in repository secrets
3. Enable GitHub Actions on your fork
4. Customize recipe.yml and trigger the Build Cloche workflow

See BlueBuild documentation
 for module reference and best practices.
